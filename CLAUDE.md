## Commands

```bash
# Run all tests
npx playwright test

# Run a specific spec file
npx playwright test tests/specs/login-flow.spec.ts

# Run a single test by name
npx playwright test --grep "Happy flow"

# Run tests headed (browser visible)
npx playwright test --headed

# Open last HTML report
npx playwright show-report

# Run only the auth setup
npx playwright test auth.setup.ts
```

## Architecture

This is a **Playwright e2e test suite** targeting the live app at `https://valentinos-magic-beans.click/`.

### Authentication flow

Tests run with a saved auth session (`playwright/.auth/user.json`). The `auth-setup` project runs first (`tests/setup/auth.setup.ts`), logs in using credentials from `playwright/.auth/loginData.json`, and saves the browser storage state. All other tests inherit this session via `storageState` in `playwright.config.ts`.

Tests that must run **unauthenticated** (e.g. login-flow, product-flow) override this with:
```ts
test.use({ storageState: { cookies: [], origins: [] } });
```

### Page Object Model

All pages live in `tests/pages/`. Each class wraps a `Page` and exposes domain-level methods using `data-test-id` selectors. Pages never import from each other.

`SignUp.ts` is an exception — it exports standalone functions instead of a class, and re-exports `signUpData` from testdata.

### Test data

- `tests/testdata/checkoutData.json` — checkout form values
- `tests/testdata/signUpData.json` — signup form values
- `playwright/.auth/loginData.json` — real login credentials (not committed; generated by `auth-flow.spec.ts`)

### Sign-up flow

`auth-flow.spec.ts` is the one-time setup test to create a new account. It uses **MailSlurp** (`EmailUtils.ts`) to create a real inbox, receive the confirmation code, and persist the new credentials to `loginData.json`. It is gated by `process.env.SIGN_UP_FLOW` and skips unless that env var is `'false'`. The API key is read from `.env` as `MAIL_SLURP_API_KEY`.

### Environment variables

| Variable | Purpose |
|---|---|
| `MAIL_SLURP_API_KEY` | MailSlurp API key for email inbox creation |
| `SIGN_UP_FLOW` | Set to `'false'` to run the sign-up test |
